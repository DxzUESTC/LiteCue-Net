# LiteCue-Net 重构指南

## 总体设计

```
视频前5秒(50帧) → 人脸对齐(224×224) → MobileNetV4(960) → Linear(960→512)
  → +CLS Token + 位置编码 → TransformerEncoder × 4(8头, FFN 2048)
  → CLS 特征 → Linear(512→2) → 分类
```
新conda环境为LitecueRefactor，当前仅安装python3.11

---

## 1. 数据采样

- 取视频前 **5 秒**，降采样到 **10fps**，共 **50 帧**
- 短于 5 秒的视频：循环填充到 50 帧
- 长于 5 秒的视频（推理时）：每 5 秒滑动窗口取 50 帧 → 所有窗口预测平均

## 2. 空间预处理

人脸裁剪对齐分为两套独立管线，分别服务于训练和推理：

### 2.1 训练预处理管线（离线）

训练前对数据集统一做一次人脸裁剪对齐，结果存为预处理后的帧序列文件。

```
原视频 → 人脸检测 + 关键点对齐 + 1.3× 边界扩放 + Resize(224×224)
  → 按帧写入 data/clips/ 目录
```

- **代码位置**: `tools/preprocess/extract_faces.py`（沿用当前实现）
- **运行时机**: 数据集构建阶段，运行一次后无需重复
- **输出**: 对齐后的人脸帧序列，后续训练 `LiteCueDataset` 直接读取

**不做额外空间增强**（无高斯模糊、色彩抖动、JPEG 压缩等）。

### 2.2 推理管线（在线）

推理时对输入视频实时做人脸检测与对齐。

```
输入视频 → 人脸检测(每帧或关键帧) → 关键点对齐 → 1.3× 边界扩放 → Resize(224×224) → 模型推理
```

- **代码位置**: `tools/analysis/video_demo_gradio.py`（沿用当前 Gradio demo 中的人脸处理逻辑）
- **运行时机**: 每次推理时实时处理
- **与训练管线的差异**: 使用同一套人脸检测器和对齐参数，确保输入分布一致

## 3. 时序预处理

- **不做帧丢弃或时序增强**
- 50 帧固定长度，不足填充

## 4. 模型架构

| 组件 | 参数 |
|------|------|
| Backbone | MobileNetV4-Small (timm, ImageNet 预训练) |
| 投影层 | Linear(960→512) |
| Positional Encoding | 可学习绝对位置编码 (50 × 512) |
| 聚合 | CLS Token |
| Transformer | 4 层 TransformerEncoder |
| 每层结构 | MultiheadAttention(8头) + FFN(512→2048→512) |
| 分类头 | Linear(512→2) |

## 5. 训练配置

- **损失函数**: Focal Loss (gamma=2.0)
- **优化器**: AdamW
  - Backbone: lr=1e-5, weight_decay=1e-4
  - Transformer + Head: lr=1e-4, weight_decay=1e-4
- **调度**: Linear warmup (3~5 epoch) → Cosine Annealing
- **批次大小**: 8~16 (24GB 显存)
- **混合精度**: FP16 AMP

## 6. 推理

- 单次 50 帧 → 前向传播 → 分类概率
- 长视频 (>5 秒): 不重叠滑动窗口 → 全窗口预测平均作为最终分数

## 9. 关键设计备忘录

### 9.1 输入输出格式

```
训练: dataset[idx] → (frames: (50, 3, 224, 224), label: int)
模型: forward(frames) → video_logits: (B, 2), feature: (B, 512)
loss: forward(video_logits, targets) → scalar
```

### 9.2 Transformer 输入构建

```
每帧: Backbone(frames[t]) → 960-dim → Linear(960→512) → 512-dim
拼接: Stack 50 帧 → (B, 50, 512)
CLS: 拼在序列头部 → (B, 51, 512)
位置编码: 可学习 51×512（CLS 也占一个位置）
输出: 取 CLS 位置的特征 → Linear(512→2)
```

### 9.3 训练/推理一致

- 推理不再启用 HRM（已删除）
- 训练和推理走完全相同的 forward 路径
- 唯一差异：推理时对 >5 秒视频做滑动窗口平均

### 9.4 滑动窗口推理（>5 秒视频）

```
窗口大小: 5 秒（50 帧）
步长: 5 秒（不重叠）
视频长度 N 秒 → N/5 个窗口 → N/5 个预测分数 → 平均
```

### 9.5 删除旧文件时机

删除操作在 Step 9 统一执行。确保先完成新代码验证再删旧文件，可随时回退。
