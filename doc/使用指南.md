## 1. 项目概述

LiteCue-Net 是一个面向 deepfake 检测的轻量化视频伪造特征拆解网络。Inter-Clip 模块使用轻量 Transformer（因果自注意力 + FFN）实现帧片段间的动态交互。

### 模型流水线

```
输入视频 (B, M, K, C, H, W)
    │
    ▼
[Stage 0] 空间特征提取 (MobileNetV4 Backbone)
    │  ┌─ 可选：频域分支 (High-Freq Residual)
    │  │   高通残差提取 → 特征图 → 在中间层(block_{fuse})残差融合
    │  └─ (B*M*K, D)
    ▼
[Stage 1] 局部微动捕捉 — Intra-Clip (DW-Conv1D)
    │  捕捉 K 帧内的瞬时运动异常
    │  → 可学习注意力池化聚合 K 帧为 Clip Token
    │  (B, M, D)
    ▼
[Stage 2] 全局不一致性分析 — Inter-Clip
    │  └─ Transformer: 因果自注意力 + FFN，内容感知动态交互
    │  (B, M, D)
    ▼
[Stage 3] 历史回顾 — HRM (推理阶段启用)
    │  利用未来信息修正过去判断 (可学习衰减系数 β)
    │  (B, M, D)
    ▼
分类头 → 可学习 clip 权重融合 → 视频级预测 + 片段级预测
                       熵正则 (防退化到 one-hot)
```

### 关键特性

| 特性 | 说明 |
|------|------|
| 轻量骨干 | MobileNetV4-Small，参数量远小于 ViT |
| 时序建模 | Transformer（因果自注意力 + FFN） |
| 频域分支 | 高通残差在 backbone 中间层融合，增强压缩鲁棒性 |
| 可学习池化 | Intra-Clip 中注意力池化替代均值池化，自动关注关键帧 |
| 可学习融合 | 视频级预测使用可学习 clip 权重 + 熵正则，抑制噪声 clip |
| 遮挡增强 | 训练期随机遮挡，提升对局部遮挡的鲁棒性 |
| 域泛化 | 支持 SupCon / CORAL / MMD / 域对抗 / GroupDRO 等多目标训练 |
| AMP | 自动混合精度训练 |
| Gradio Demo | 提供交互式 Web 演示界面 |

## 5. 配置说明

### 5.1 配置结构

训练配置通过 YAML 文件组织，主配置入口为 `configs/train.yaml`，通过 `dataset_config` 字段引用数据集配置。

```
train.yaml
  ├── dataset_config ───→ dataset/faceforensics.yaml
  │                        (数据路径、过滤规则、划分比例)
  ├── 训练超参数 (epochs, batch_size, lr, ...)
  ├── 模型参数 (feature_dim, clip_num, clip_len, backbone, ...)
  ├── 增强配置 (augmentation, occlusion, domain_randomization)
  ├── 采样配置 (balanced sampling)
  └── 损失与泛化 (loss, generalization)
```

### 5.2 核心参数速查

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `model.clip_num` | 16 | 全局片段数 M |
| `model.clip_len` | 4 | 片段内帧数 K |
| `model.feature_dim` | 256 | 内部特征维度 D |
| `model.backbone` | mobilenetv4_conv_small.e2400_r224_in1k | 骨干网络 |
| `model.use_temporal_diff` | false | 是否启用显式时序差分 |
| `model.use_frequency_branch` | true | 是否启用频域分支 |
| `model.frequency_fuse_block` | 2 | 频域分支融合位置 (0~4 backbone block 索引)，-1 退化为原始行为 |
| `model.temporal_module` | attention | Inter-Clip 时序建模方式：`attention` (Transformer, 默认) / `gated_mlp` (已弃用) |
| `model.token_dropout` | 0.0 | >0 时训练期随机丢弃部分 clip token，提升时序鲁棒性 |
| `loss.video_weight` | 1.0 | 视频级损失权重 |
| `loss.clip_weight` | 1.0 | 片段级损失权重 |
| `loss.focal_alpha` | 0.25 | Focal Loss α（应对样本不平衡） |
| `loss.focal_gamma` | 2.0 | Focal Loss γ（聚焦难分样本） |
| `loss.entropy_weight` | 0.01 | clip 权重熵正则强度，防止 softmax 退化到 one-hot |
| `occlusion.enabled` | true | 是否启用遮挡增强 |
| `generalization.enabled` | false | 是否启用域泛化训练 |

### 5.3 域泛化配置

`configs/train_generalization.yaml` 提供了完整的域泛化训练预设，支持以下技术：

| 技术 | 参数 | 说明 |
|------|------|------|
| SupCon | `generalization.supcon_weight` | 监督对比学习 |
| CORAL | `generalization.coral_weight` | 相关性对齐 |
| MMD | `generalization.mmd_weight` | 最大均值差异 |
| 域对抗 | `generalization.domain_adv_weight` | 对抗性域分类器 |
| GroupDRO | `generalization.groupdro_weight` | 分组鲁棒优化 |

## 6. 训练模型

### 6.1 基本训练

```bash
# 使用默认配置训练
python main.py --config configs/train.yaml
```

### 6.2 域泛化训练

```bash
# 使用域泛化配置
python main.py --config configs/train_generalization.yaml
```

### 6.3 训练流程说明

1. **数据加载**：根据索引文件加载数据
2. **身份隔离划分**：基于身份 ID 将数据划分为 train/val/test，防止身份泄露
3. **训练循环**：
   - 每个 epoch 遍历训练集，支持 AMP 混合精度
   - 训练日志输出：Loss / LossV / LossC / Acc / LR
   - 每个 epoch 后验证集评估（AUC / ACC / AP / EER / TPR@1%FPR / 混淆矩阵）
   - 保存最佳模型（按 AUC）到 `save_dir/best_model.pth`
4. **测试**：训练完成后在测试集上最终评估

### 6.4 训练输出

```
checkpoints/exp_20260305/
├── best_model.pth          # 验证集最优权重
├── last_epoch.pth          # 最后一轮权重
└── checkpoint_epoch_XX.pth # 中间检查点

logs/
└── train_exp_20260305_20260305_143022.log  # 训练日志
```

### 7.4 评估指标

- **AUC**：ROC 曲线下面积（主要指标）
- **ACC**：准确率
- **AP**：平均精度 (Average Precision)
- **EER**：等错误率 (Equal Error Rate)
- **TPR@FPR=1%** / **TPR@FPR=0.1%**：低误报率下的检出率
- **混淆矩阵**：TN / FP / FN / TP 明细
- 支持按数据集、按伪造方法分层统计

## 8. 推理与部署

### 8.1 Gradio Web Demo

项目提供了基于 Gradio 的交互式 Web 界面，支持上传视频/图片进行实时检测。当前版本支持所有新模型参数（频域分支、时序模块选择等）。视频判定为伪造时，Demo 会按 clip 伪造概率选取关键帧，并输出叠加 Grad-CAM 的关键帧图片集，用于提示模型关注的可疑区域。

```bash
# 启动 Gradio 服务（需要先准备好模型权重）
python tools/analysis/video_demo_gradio.py
```

### 8.2 FastAPI 推理服务

提供 RESTful API 接口，支持视频上传、实时鉴伪推理和 Grad-CAM 热力图可视化。

```bash
# 安装推理服务依赖
pip install -r requirements-api.txt

# （可选）GPU 加速人脸检测
pip install onnxruntime-gpu

# 启动服务
python api/main.py
# 或 uvicorn api.main:app --host 0.0.0.0 --port 8001 --reload
```

**架构特点：**
- **Core Layer** (`api/core/`)：RetinaFace ONNX 人脸检测 + 5-pt landmark 仿射对齐，已剥离 InsightFace 依赖
- **Model Layer** (`api/model/`)：LiteCueNet 网络定义（推理独立副本）
- **Backend Layer** (`api/backends/`)：推理引擎抽象层，自动从 checkpoint 推断架构参数，集成 Grad-CAM

**API 端点：**
- `GET /api/v1/health` — 健康检查
- `POST /api/v1/detect` — 上传视频进行鉴伪，返回伪造概率 + Grad-CAM 热力图

详细文档请参考 [LiteCue-Net FastAPI 推理服务.md](LiteCue-Net%20FastAPI%20推理服务.md)。

### 8.3 代码调用推理

```python
import torch
from src.models.detector import LiteCueNet

# 加载模型（支持新参数）
model = LiteCueNet(
    feature_dim=256, clip_num=16, clip_len=4, num_classes=2,
    use_frequency_branch=True,           # 启用频域分支
    frequency_fuse_block=2,              # 频域特征融合位置
    temporal_module="attention",          # Transformer（因果自注意力 + FFN）
)
checkpoint = torch.load("checkpoints/exp_20260511/best_model.pth", map_location="cpu", weights_only=True)
model.load_state_dict(checkpoint["state_dict"], strict=False)
model.eval()

# 推理（输入形状: B, M, K, C, H, W）
video_tensor = torch.randn(1, 16, 4, 3, 224, 224)
with torch.no_grad():
    video_logits, clip_logits = model(video_tensor)
    pred = video_logits.argmax(dim=1)
```

## 9. 常见问题

**Q: 训练时报显存不足？**  
A: 降低 `batch_size` 和/或 `model.clip_num` 的值；也可以关闭 `use_amp: false`。

**Q: 数据加载报错 `FileNotFoundError`？**  
A: 检查 `configs/dataset/*.yaml` 中的 `data_root` 和 `index_path` 是否与实际路径一致。

**Q: 跨平台路径问题？**  
A: 项目内部统一使用 `os.path.join` 处理路径；配置文件中建议使用 `/` 作为分隔符。

**Q: 如何在自己的数据集上训练？**  
A: 参考 [数据集预处理指南](LiteCue-Net%20数据集预处理指南.md)，按格式准备 clip 数据并构建索引文件，然后在 `configs/dataset/` 下添加新的数据集配置。
